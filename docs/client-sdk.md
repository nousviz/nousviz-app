# Client SDK

NousViz ships auto-generated client SDKs for **TypeScript** and **Python**.
Both are derived from the live `/openapi.json` — the same spec that
powers `/docs/api`'s schema panels and try-it-out widget.

> **Distinct from `nousviz_sdk`** (the [Plugin SDK](sdk-reference.md) that
> runs *inside* a NousViz instance). The client SDKs are for **external
> integrations** that talk to a NousViz instance over HTTP.

## TypeScript — `@nousviz/client`

### Install

```bash
npm install @nousviz/client
```

### Three-line example

```ts
import { createClient, AuthService, PluginsService } from "@nousviz/client";

createClient({ baseUrl: "https://nousviz.online", getToken: () => localStorage.getItem("token") ?? "" });

const me = await AuthService.authMe();
const plugins = await PluginsService.pluginsList();
```

### Auth

`createClient({ getToken })` wires the shared client to read your
session token at request time. Public endpoints (`AuthService.authStatus()`,
`HealthService.healthCheck()`) work without a token.

To issue a token:

```ts
import { AuthService } from "@nousviz/client";

const resp = await AuthService.authLogin({
  requestBody: { email: "you@example.com", password: "..." },
});
localStorage.setItem("token", resp.token);
```

### Operation naming

Method names follow the platform's `<resource>.<verb>` `operationId`
(B217). The TypeScript generator camelCases them:

| HTTP | Method |
|---|---|
| `GET /api/auth/me` | `AuthService.authMe()` |
| `GET /api/plugins` | `PluginsService.pluginsList()` |
| `POST /api/plugins/{id}/install` | `PluginsService.pluginsInstall({ pluginId, requestBody })` |
| `GET /api/system/permissions` | `SystemService.systemPermissions()` |

## Python — `nousviz-client`

### Install

```bash
pip install nousviz-client
```

### Three-line example

```python
from nousviz_client import AuthenticatedClient
from nousviz_client.api.auth import auth_me
from nousviz_client.api.plugins import plugins_list

client = AuthenticatedClient(base_url="https://nousviz.online", token="<your-session-token>")
me = auth_me.sync(client=client)
plugins = plugins_list.sync(client=client)
```

### Auth

`AuthenticatedClient(token=...)` sets the `X-Session-Token` header on
every request.

To issue a token:

```python
from nousviz_client import Client
from nousviz_client.api.auth import auth_login
from nousviz_client.models import LoginRequest

client = Client(base_url="https://nousviz.online")
resp = auth_login.sync(client=client, body=LoginRequest(email="you@example.com", password="..."))
print(resp.token)
```

### Sync vs async

Each operation module exposes four functions:
- `sync(client, ...)` — synchronous, returns the parsed body.
- `asyncio(client, ...)` — async via `httpx.AsyncClient`.
- `sync_detailed(client, ...)` / `asyncio_detailed(client, ...)` —
  return the full `Response[T]` wrapper with status + headers.

```python
from nousviz_client.api.health import health_check

resp = health_check.sync_detailed(client=client)
print(resp.status_code)         # → 200
print(resp.parsed.status)       # → "healthy"
```

## Generation

Both SDKs are generated by `scripts/regenerate-clients.sh` (run from
the repo root). The script:

1. Dumps the live spec via `app.openapi()`.
2. Regenerates `packages/client-ts/src/generated/` via
   `openapi-typescript-codegen`.
3. Regenerates `packages/client-py/nousviz_client/` via
   `openapi-python-client`.
4. Bumps both packages' versions to match the platform `VERSION` file.

The generated source is **committed to the repo** so spec changes
produce a diffable PR. The `tests/test_openapi_stability.py` snapshot
guarantees the live spec matches the committed snapshot; the `npm run
build` and `python -m build --wheel` checks guarantee the generated
SDKs compile.

## Versioning

Both SDKs ship at the same version as the platform. Patch releases of
the platform regenerate the SDKs at the matching version. Breaking
changes to the spec (rare; gated by the snapshot test) increment the
minor version.
