# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please report vulnerabilities via one of:

1. **GitHub Private Vulnerability Reporting:**
   Go to the [Security tab](../../security/advisories/new) of this repository
   and click "Report a vulnerability".

2. **Email:** security@nousviz.com

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Impact assessment (if known)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 5 business days
- **Fix for critical issues:** Within 14 days

We will credit reporters in the release notes unless they prefer anonymity.
